# FL Pulse — FL Studio Plugin Sound Board

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/ceskare/web_plugin)

Веб-сервис для мгновенного прослушивания оригинального звучания плагинов FL Studio. Пользователь вводит название плагина, и сервис автоматически находит и воспроизводит оригинальное аудио-превью (из YouTube, SoundCloud или загруженных автором файлов).

## Особенности проекта

1. **Динамический поиск и проверка релевантности:** Автоматический поиск на YouTube и SoundCloud без использования сторонних API-ключей. Умный алгоритм «на лету» сверяет ключевые слова запроса пользователя с заголовками видео/треков, отсекая нерелевантный мусор. Если плагин редкий или введен с ошибкой, сервис корректно сообщит, что звук не найден.
2. **Безопасное и легковесное хранение:** Метаданные загруженных треков и база плагинов хранятся в надежной, транзакционной SQLite базе данных (файл `plugins.db`), которая не требует настройки сервера и потребляет 0 МБ ОЗУ при простое.
3. **Модерация загрузок:** Простая капча для защиты от ботов на форме загрузки и удобная закрытая админ-панель модератора по адресу `/admin.html` (доступ по паролю) для одобрения и удаления файлов.
4. **Стильный Pitchfork-дизайн:** Черно-бело-красная эстетика в стиле Spotify и Pitchfork с использованием HTML5 Canvas-визуализатора частот для воспроизводимого аудио.

## Как запустить локально

### Шаг 1. Настройка виртуального окружения
Убедитесь, что у вас установлен Python 3.10+ и менеджер пакетов `uv` (или стандартный `pip`).

```bash
# Создание виртуального окружения
uv venv

# Активация виртуального окружения (Windows)
.venv\Scripts\activate

# Активация виртуального окружения (Linux/macOS)
source .venv/bin/activate
```

### Шаг 2. Установка зависимостей
```bash
uv pip install -r requirements.txt
# Или через обычный pip:
# pip install -r requirements.txt
```

### Шаг 3. Запуск веб-сервера
```bash
python -m uvicorn backend.main:app --reload
```
После запуска откройте в браузере: **http://localhost:8000**

* Чтобы войти в панель модерации (http://localhost:8000/admin.html), используйте стандартный пароль: `admin`.

### Шаг 4. Запуск автоматических тестов
Тесты написаны на pytest и проверяют работу капчи, алгоритм фильтрации релевантности и полный цикл загрузки/модерации файлов:
```bash
pytest tests/
```

## Как развернуть в интернете бесплатно (Хостинг)

Так как приложению требуется бэкенд на Python (для динамического поиска на YouTube/SoundCloud и загрузки файлов), его **нельзя развернуть напрямую на GitHub Pages** (GitHub Pages поддерживает только статические HTML/CSS/JS сайты).

Для бесплатного размещения приложения с бэкендом и базой данных есть несколько отличных альтернатив:

### Вариант 1. Render (Рекомендуемый)
Render — это современная и бесплатная платформа для хостинга веб-приложений.
1. Создайте бесплатный аккаунт на [Render.com](https://render.com/).
2. Создайте новый репозиторий на вашем GitHub и загрузите туда код проекта.
3. На Render нажмите **New** -> **Web Service** и подключите ваш репозиторий с GitHub.
4. Заполните настройки проекта:
   - **Environment:** `Python`
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python -m uvicorn backend.main:app --host 0.0.0.0 --port $PORT`
5. Нажмите **Deploy**. Через 2 минуты проект будет доступен по ссылке вида `https://fl-pulse.onrender.com`.

*Примечание:* Для изменения пароля администратора в настройках Render добавьте переменную окружения (Environment Variable): `ADMIN_PASSWORD = ваш_новый_пароль`.

### Вариант 2. Hugging Face Spaces (Бесплатно и 24/7)
Hugging Face бесплатно предоставляет серверы для Python-приложений.
1. Создайте аккаунт на [Hugging Face](https://huggingface.co/).
2. Перейдите во вкладку **Spaces** -> **Create new Space**.
3. Выберите Space SDK: **Docker** (шаблон Blank).
4. Создайте файл `Dockerfile` в корне проекта (пример ниже) и загрузите все файлы проекта в созданный Space.
5. Hugging Face автоматически соберет проект и запустит его.

#### Пример Dockerfile для Hugging Face:
```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 7860
CMD ["python", "-m", "uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "7860"]
```
После сборки вы получите прямую ссылку, которую можно отправить друзьям.
